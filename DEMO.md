# DEMO 腳本 — AI 政策輔助報告（港務視角）

競賽現場 demo 用。核心敘事：**以港務工作人員視角**——現場遇到問題即時查、長官寫報告時直接參考生成草稿再改。每個回答都 grounded 於知識庫、附來源引用、可追溯。

---

## 0. 前置檢查（demo 前務必跑一次）

| 項目 | 指令 / 動作 | 確認 |
|---|---|---|
| 資料庫 | `docker start imarine-pg` | pgvector 在 :5544 |
| 後端 | `cd iMarine-rag-backend && uv run uvicorn src.rag_agent.main:app --port 8100` | `/health` 回 ok |
| **知識庫已灌** | 首次或新環境：`curl -X POST http://localhost:8100/api/ingest/run` | **8 個來源都有 chunks（見下）** |
| 前端 | `cd iMarine-FrontEnd && npm run dev` | http://localhost:5173 |

> ⚠️ **最重要**：知識庫沒 ingest 過的話，報告會因「證據不足」而很爛。確認 `GET /api/sources` 每個來源都有 chunk 數（商港法 76、替代能源各庫、新聞…共 ~489），不是只有新聞。
> 快速自檢：`uv run python scripts/eval_report.py`（6 情境應全過）。

進頁：http://localhost:5173 → 左側選「AI 政策輔助報告」→ 預設停在「綜合對話 · 全部來源」。

## 視角一 — 港務現場人員：遇到問題即時查

**操作**：綜合對話輸入框打問題 → 送出 → 秒回，答案附商港法出處、右欄來源連動。
**話術**：「現場人員遇到狀況，不用翻法規全文，問系統就好，而且答案追溯得到商港法第幾條。」

複製即用（已實測命中商港法）：

- **船舶要進出商港需要辦理哪些手續？** ⭐
- **碼頭裝卸作業的安全管理，法規上有什麼要求？** ⭐
- **港區發生船舶油污染，商港法規定該怎麼處理？**
- **商港服務費是怎麼規定的？哪些船舶要繳？**

## 視角二 — 長官寫報告：依過去資料生草稿再改

**操作**：在「綜合對話 · 全部來源」→ 輸入框打報告需求 → 下方「產報告」列選模版（預設「政策輔助報告（四章節）」）→ 按「產生報告」。來源用右欄勾選的（預設全勾）。
**話術**：「長官要寫政策報告，不用從零開始——系統依知識庫（法規＋政策＋新聞）生出**結構化草稿＋逐段引用**，長官在這基礎上改。」

複製即用（已 eval 驗證，覆蓋率高）：

- **擬一份「IMO 淨零框架對我國港埠與航商的影響與因應」政策參考報告** ⭐（覆蓋 100%）
- **擬一份「推動港口岸電」的政策說明報告，供長官參考**（92%）
- **擬一份「商港法對港務作業規範」的重點整理報告**（92%）

五個模版（下拉可選）：

- **`航港政策研究報告（六段）`** ⭐ demo 首選——**仿交通部運研所真實委託研究報告**：前言／國際規範趨勢／國際港口案例／我國現況／課題與挑戰／結論建議。最貼近長官習慣的報告格式。
- **`國際海事動態分析（五段）`**——仿航港局海事公約動態分析：國際要聞／重點會議議題／對我國影響／建議事項／後續追蹤。適合定期情報彙整。
- `政策輔助報告（四章節）`＝背景／政策法源／國際案例／建議事項（預設）。
- `焦點摘要（速報）`、`自由格式（依需求）`。

> demo 建議選 **`航港政策研究報告（六段）`**：現場秀「這章節架構是照航港局真實委託研究報告做的」，說服力最強。

**產完報告可匯出**：報告卡下方有「下載 Markdown」（可帶走編輯）與「列印 / 存 PDF」兩顆按鈕 → 現場秀「長官直接下載、改成自己的版本」。

## 視角三（開場鉤子）— 每日情報彙整

**操作**：點左側收件匣頂部「MM-DD 海運晨報」→ 中間顯示條列重點＋來源引用 → 點下方建議問題 chip 現場問 → 按「更新新聞」重抓重生成。
**話術**：「系統每天從航港局新聞知識庫**即時生成**海運晨報，不是寫死的。」
**可設定自動時間**：系統設定 → 政策報告 →「新聞自動更新」可**設定每天幾點自動抓新聞、重生成晨報**（例：每天 06:30）→ 現場秀「不用人工，每天上班前情報就備好了」。demo 時可開啟並設個近的時間，或用「立即更新一次」即時展示。

## 建議動線（~3 分鐘，三幕）

1. **開場（晨報）**：點「海運晨報」→ 按「更新新聞」→「每天自動彙整情報、可再追問」。
2. **現場視角**：綜合對話問「**船舶要進出商港需要辦理哪些手續？**」→ 秒回附商港法出處 →「現場即時查、可追溯」。
3. **長官視角**：輸入「**IMO 淨零框架…政策參考報告**」→ 選政策輔助報告模版 → 產生 → 展示四章節、逐句 `[n]` 引用連動右欄、Grounding 100% →「長官寫報告直接參考改」。

收尾金句：「**每一句都有據可查、可追溯到來源，不是 LLM 亂編。**」

## 備用 prompt

現場查詢：`引水（領港）作業的法規要求是什麼？`、`商港區域內哪些行為被禁止、罰則為何？`
產報告：`擬一份「替代燃料加注基礎設施推動」建議報告`、`台灣港口綠色甲醇加注的政策現況與建議`
對話：`哪些國際港口已具備替代燃料的商業化加注能力？`、`台灣有哪些航商已導入 LNG 雙燃料船？`

## 現場注意 / 疑難排解

- **後端沒起**：右欄會 fallback 顯示 mock 假來源、提問回「後端連線失敗」。→ 確認 :8100。
- **報告很爛 / 證據不足**：知識庫沒 ingest。→ 跑 `POST /api/ingest/run`。
- **雲端模型偶爾逾時**：NCHC 端點尖峰會慢，重送一次即可。demo 前先各跑一次暖機。
- **地端/雲端切換**：切「地端」需真的有跑 Ollama，否則 toast 提示、後端維持雲端（刻意的安全守衛）。demo 全程用雲端即可。
- 模型設定在「系統設定 → 政策報告 → 模型管理」：可用「+ 自訂供應商」填雲端 URL/KEY/模型 id、測試連線、儲存。

## 待強化（非 demo 阻礙，記錄用）

- **報告模版取自真實過去報告**：目前三個模版的章節結構是內建的通用版。下一步要**蒐集航港局過去的實際報告，萃取其章節結構做成模版**，讓產出更貼近長官習慣的格式。相關：知識庫管理支援上傳文件（TXT/MD/PDF/DOCX），過去報告也可上傳進知識庫當檢索參考。
- 部分章節「單一來源撐整段」（見 `docs/report-scenarios.md` 於後端 repo），可在檢索/prompt 層強化證據多樣性。
